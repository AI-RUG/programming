Exercises

Exercise 1: Hello World

If you have dabbled in programming before, you may be familiar with the “Hello World” program. This is a small program that prints “Hello world” to the terminal. If you have programmed and/or written a hello world program before, it is okay to skip this exercise.

Try to write a program in Python that prints the text “Hello World'' to the terminal.

Exercise 2: Hello [name]

There's nothing quite like being greeted.

Write a program that asks for your name, and then greets you using your name.

Exercise 3: Hello Alice and Bob

Today, I have decided that I only want to greet Alice and Bob.

Write a program that only greets users called Alice or Bob.

Exercise 4: Very odd

At this point, I hope you know how to tell whether a number is even or odd. However, sometimes it's just easier to be able to ask someone (or in this case, something) else.

Write a program that takes a number as input, and prints whether that number is even or odd.

Exercise 5: Give me sum

Sometimes it is useful to calculate the sum of the numbers 1 to n. For n = 3 this sum would be 1 + 2 + 3 = 6. For small numbers it is not very difficult to calculate this yourself, but for large numbers it can take some time.

Write a program that asks the user for a number n and prints the sum of the numbers 1 to n.

Exercise 6: Multiples of 3 and 5 only

The previous program can be altered in many ways. You may only like multiples of certain numbers. I only like multiples of 3 and 5, such as 3, 5, 6, 9, 10, etc.

Modify the previous program such that it only sums the multiples of three or five in the range of i to n.

Exercise 7: Product or sum?

We would like to continue building on the previous program. Sometimes, you may need the factorial instead of the sum. The factorial of a number n is the product of numbers 1 until n.

Write a program that asks the user for a number n and gives them the possibility to choose between computing the sum and computing the product of 1,...,n.

Exercise 8: Guessing game

This next exercise shows you a more practical application of things you have used so far. It's a game! Because this game has some room for creative freedom, we will leave it up to you whether the user can specify a range or whether there is a standard range. We also have not talked about libraries yet. You will learn about that if you choose to study AI. For now, follow the following instructions if you want to generate a random number in a certain range in Python:

  1. Put import random all the way at the top of your code. It needs to be the very first thing in your file.
  2. If you want to generate a number between x and y, type random.randint(x, y). This range is inclusive, which means that both x and y could be the random number.

Write a guessing game where the user has to guess a secret number. After every guess the program tells the user whether their number was too large or too small. At the end the number of tries needed should be printed.

Exercise 9: Leap years ahead

Some things are more complex than they look. The problem below is one of those things. While solving it, do not just increment the first leap year you find by 4. Please make sure to check whether something is a leap year by using the rules for when something is a leap year. If you do not know those by heart (it’s more complex than it seems), Google it!

Write a program that takes a year as an input and that prints both whether that year is a leap year, and the next 20 leap years.

Exercise 10: Raise a glass!

Surprise! To celebrate the end of this practical, we’d like you to try solving a programming exercise from the first week of the AI first-year course ‘Imperative Programming’. The phrasing is exactly the same as in the original assignment. Good luck!

John wants to celebrate that he has passed the course ’Imperative Programming’ with his friends. The plan is to serve everyone (including himself) one glass with 150ml of champagne. John found a website where you can order champagne per Magnum bottle (containing 1.5 litres)

Write a program that reads from the input the number of invited friends (a positive integer) and outputs the number of bottles that John needs to order.
